File Systems

A file system is the method and data structure that an operating system uses to manage files on a storage device. It defines how data is stored, accessed, organized, and secured.

1. Objectives of a File System

  • Efficient storage and retrieval of data
  • Logical organization of files and directories
  • Access control and security
  • Support for metadata and file attributes
  • Fault tolerance and recovery mechanisms

2. File Concepts

  • File: A named collection of related data
  • File Attributes: Name, type, size, creation date, permissions
  • File Operations: Create, open, read, write, delete, rename, close

3. Directory Structure

  • Single-Level: All files in one directory
  • Two-Level: Separate user directories
  • Tree Structure: Hierarchical directories and subdirectories
  • Acyclic Graph: Allows shared files via links

4. File Access Methods

  • Sequential Access: Data read in order; suitable for tapes
  • Direct Access: Access any block directly; used in disks
  • Indexed Access: Uses index table for fast lookup

5. File Allocation Methods

5.1 Contiguous Allocation

Each file occupies a set of contiguous blocks. Simple and fast but suffers from external fragmentation.

File A: Blocks 5–14
File B: Blocks 15–20

5.2 Linked Allocation

Each file is a linked list of blocks. No fragmentation, but slow random access.

File A: Block 5 → Block 9 → Block 12 → ...

5.3 Indexed Allocation

Each file has an index block containing pointers to its data blocks. Supports direct access and avoids fragmentation.

Index Block for File A: [5, 9, 12, 20]

6. Free Space Management

  • Bitmap: Each bit represents a block (0 = free, 1 = used)
  • Linked List: Free blocks linked together
  • Grouping: Stores addresses of free blocks in groups
  • Counting: Stores starting block and number of free blocks

7. File System Mounting

Mounting makes a file system accessible to the OS. It involves verifying the file system structure and integrating it into the directory tree.

8. File Protection

  • Access Control: Read, write, execute permissions per user/group
  • Encryption: Protects file contents from unauthorized access
  • Backup and Recovery: Ensures data integrity and availability

9. Journaling and Reliability

Modern file systems use journaling to log changes before applying them. This helps recover from crashes and ensures consistency.

10. Common File Systems

File System Platform Features
FAT32 Windows, Embedded Simple, widely supported, no journaling
NTFS Windows Journaling, permissions, compression
ext4 Linux Journaling, large file support, backward compatible
XFS Linux High performance, scalable, journaling
APFS macOS Snapshots, encryption, space sharing

11. Virtual File Systems (VFS)

VFS provides an abstraction layer that allows multiple file systems to coexist. It enables uniform access to different file system types.

12. File System in Distributed Systems

  • Network File System (NFS): Allows remote file access over network
  • Google File System (GFS): Designed for large-scale data processing
  • HDFS: Hadoop Distributed File System for big data
