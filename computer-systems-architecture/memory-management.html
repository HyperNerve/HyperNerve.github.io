Memory Management

Memory management is a core function of the operating system that handles the allocation, tracking, and optimization of system memory. It ensures efficient use of RAM and provides isolation between processes.

1. Objectives

  • Efficient memory allocation and deallocation
  • Protection and isolation between processes
  • Support for multitasking and virtual memory
  • Minimization of fragmentation

2. Types of Memory

  • Primary Memory: RAM, cache
  • Secondary Memory: HDD, SSD
  • Virtual Memory: Disk space used as RAM extension

3. Memory Allocation Techniques

  • Contiguous Allocation: Each process gets a single continuous block
  • Non-Contiguous Allocation: Memory divided into segments or pages
  • Dynamic Allocation: Memory assigned at runtime

4. Paging

  • Divides memory into fixed-size pages
  • Uses page tables to map virtual to physical addresses
  • Eliminates external fragmentation

5. Segmentation

  • Divides memory into variable-sized segments
  • Each segment has a base and limit
  • Supports logical division (code, data, stack)

6. Virtual Memory

  • Allows execution of processes larger than physical memory
  • Uses disk as an extension of RAM
  • Implemented via paging and swapping

7. Fragmentation

  • External Fragmentation: Free memory scattered in small blocks
  • Internal Fragmentation: Unused space within allocated blocks

8. Memory Protection

  • Prevents unauthorized access between processes
  • Uses base and limit registers or page-level permissions

9. Swapping

  • Temporarily moves inactive processes to disk
  • Frees up RAM for active processes

10. Allocation Algorithms

  • First Fit: Allocates first suitable block
  • Best Fit: Allocates smallest suitable block
  • Worst Fit: Allocates largest available block
