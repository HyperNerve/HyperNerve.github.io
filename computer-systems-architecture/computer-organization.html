Computer Organization

Computer Organization refers to the operational structure and physical implementation of a computer system. It focuses on how hardware components interact to execute instructions, manage data, and perform computation.

1. Functional Units of a Computer

  • Input Unit: Devices that send data to the system (e.g., keyboard, mouse)
  • Output Unit: Devices that receive data from the system (e.g., monitor, printer)
  • Memory Unit: Stores data and instructions
  • Arithmetic Logic Unit (ALU): Performs arithmetic and logical operations
  • Control Unit (CU): Directs operations of the processor and coordinates components

2. CPU Structure

  • Registers: Small, fast storage locations inside the CPU
  • Program Counter (PC): Holds address of next instruction
  • Instruction Register (IR): Holds current instruction
  • Accumulator (AC): Temporary storage for arithmetic results
  • Flags: Status indicators (zero, carry, overflow)

3. Instruction Cycle

  • Fetch: Retrieve instruction from memory
  • Decode: Interpret the instruction
  • Execute: Perform the operation
  • Store: Save result to memory or register

4. Bus Architecture

  • Data Bus: Transfers actual data
  • Address Bus: Carries memory addresses
  • Control Bus: Carries control signals (read/write)

5. Memory Organization

  • Main Memory (RAM): Volatile, fast access
  • ROM: Non-volatile, stores firmware
  • Cache: Small, fast memory between CPU and RAM
  • Registers: Fastest memory, inside CPU

6. I/O Organization

  • Programmed I/O: CPU polls devices
  • Interrupt-driven I/O: Devices signal CPU when ready
  • DMA (Direct Memory Access): Devices transfer data directly to memory

7. Instruction Formats

  • Zero-address: Stack-based
  • One-address: Accumulator-based
  • Two-address: Source and destination
  • Three-address: Two operands and result location

8. Addressing Modes

  • Immediate: Operand is part of instruction
  • Direct: Address of operand is given
  • Indirect: Address points to another address
  • Register: Operand is in a register
  • Indexed: Base address + index

9. Control Unit Design

  • Hardwired Control: Fixed logic circuits
  • Microprogrammed Control: Uses control memory and microinstructions

10. Performance Factors

  • Clock Speed: Determines instruction rate
  • Instruction Set Efficiency: Affects execution time
  • Memory Access Time: Influences overall speed
  • Bus Width: Determines data transfer capacity

11. Real-World Examples

  • Intel x86: CISC architecture with complex instruction formats
  • ARM: RISC architecture used in mobile and embedded systems
  • RISC-V: Open-source instruction set gaining popularity

12. Applications

  • Embedded Systems: Efficient control logic and I/O handling
  • Operating Systems: Resource management and interrupt handling
  • Compiler Design: Instruction selection and register allocation
