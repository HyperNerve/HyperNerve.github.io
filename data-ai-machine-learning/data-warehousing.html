Data Warehousing and OLAP

Data warehousing is the process of collecting, storing, and managing large volumes of structured data for analysis and reporting. OLAP (Online Analytical Processing) enables multidimensional analysis of this data to support decision-making.

1. What is a Data Warehouse?

  • Central Repository: Stores integrated data from multiple sources
  • Historical Data: Optimized for read-heavy analytical queries
  • ETL Process: Extract, Transform, Load pipeline used to populate warehouse

2. Characteristics of Data Warehouses

  • Subject-Oriented: Organized around key business areas
  • Time-Variant: Maintains historical snapshots
  • Non-Volatile: Data is stable and not frequently updated

3. OLAP Overview

  • Multidimensional Analysis: View data across multiple dimensions (e.g., time, region, product)
  • Operations: Roll-up, drill-down, slice, dice, pivot
  • OLAP Cubes: Pre-aggregated data structures for fast querying

4. OLAP Operations

Operation Description
Roll-up Aggregate data (e.g., daily → monthly)
Drill-down View more detailed data (e.g., yearly → quarterly)
Slice Filter by one dimension (e.g., region = 'Asia')
Dice Filter by multiple dimensions
Pivot Reorient dimensions for different views

5. ETL Pipeline Example

Extract: Pull data from CRM, ERP, logs
Transform: Clean, normalize, enrich
Load: Insert into warehouse tables

6. Data Warehouse Architectures

  • Top-down: Centralized warehouse with dependent data marts
  • Bottom-up: Independent data marts integrated into warehouse
  • Hybrid: Combines both approaches

7. OLAP Types

  • MOLAP: Multidimensional OLAP (pre-computed cubes)
  • ROLAP: Relational OLAP (queries relational tables)
  • HOLAP: Hybrid OLAP (combines MOLAP and ROLAP)

8. Tools & Platforms

  • Data Warehousing: Amazon Redshift, Snowflake, Google BigQuery
  • OLAP Engines: Apache Kylin, Microsoft SSAS, SAP BW
