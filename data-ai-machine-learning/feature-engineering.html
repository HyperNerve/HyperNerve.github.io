Feature Engineering

Feature Engineering is the process of selecting, transforming, and creating input variables (features) that improve the performance of machine learning models. It bridges raw data and predictive power.

1. Goals of Feature Engineering

  • Improve model accuracy and generalization
  • Reduce noise and irrelevant data
  • Make patterns more detectable by algorithms

2. Common Techniques

  • Missing Value Handling
    • Imputation (mean, median, mode)
    • Flag missing values as separate feature
  • Encoding Categorical Variables
    • One-hot encoding
    • Label encoding
    • Target encoding
  • Scaling and Normalization
    • Min-Max scaling
    • Standardization (Z-score)
  • Feature Creation
    • Combine existing features (e.g., price Ã— quantity = revenue)
    • Extract date parts (e.g., day, month, weekday)
    • Text features (e.g., word count, sentiment score)
  • Dimensionality Reduction
    • Principal Component Analysis (PCA)
    • Feature selection based on importance

3. Example: Transaction Data

Raw Features:
  - Date: "2025-08-10"
  - Amount: 1200
  - Category: "Electronics"

Engineered Features:
  - Month: 8
  - Weekday: Sunday
  - IsHighValue: True
  - Category_Electronics: 1

4. Feature Selection Methods

  • Filter methods (e.g., correlation, chi-square)
  • Wrapper methods (e.g., recursive feature elimination)
  • Embedded methods (e.g., regularization, tree-based importance)

5. Tools & Libraries

  • Python: pandas, scikit-learn, Feature-engine
  • R: caret, dplyr
  • AutoML platforms: H2O, DataRobot

6. Challenges

  • Overengineering features that lead to overfitting
  • Data leakage from future information
  • High dimensionality and redundancy

7. Applications

  • Finance: Risk scoring, fraud detection
  • Retail: Customer segmentation, demand forecasting
  • Healthcare: Diagnostic prediction, treatment recommendation
