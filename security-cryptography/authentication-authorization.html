Authentication and Authorization

Authentication and Authorization are foundational concepts in security. Authentication verifies user identity, while authorization determines what resources a user is allowed to access after being authenticated.

1. Objectives

  • Ensure only legitimate users can access systems
  • Control access to sensitive resources
  • Prevent privilege escalation and misuse

2. Authentication Methods

  • Passwords: Traditional credential-based login
  • Biometrics: Fingerprint, face, voice recognition
  • Tokens: One-time passwords (OTP), hardware keys
  • Multi-Factor Authentication (MFA): Combination of methods

3. Authorization Models

  • Role-Based Access Control (RBAC): Permissions based on user roles
  • Attribute-Based Access Control (ABAC): Rules based on user attributes
  • Discretionary Access Control (DAC): Resource owners define access
  • Mandatory Access Control (MAC): System-enforced policies

4. Example: Web Application Login

User enters username and password

System verifies credentials using hash comparison

If valid, user is authenticated and assigned role

Role determines access to dashboard, settings, or admin panel

5. Session Management

  • Use secure session tokens
  • Set expiration and renewal policies
  • Store tokens securely (e.g., HttpOnly cookies)

6. Identity Providers & Protocols

  • OAuth 2.0 (delegated access)
  • OpenID Connect (authentication layer)
  • SAML (enterprise identity federation)
  • LDAP (directory-based authentication)

7. Best Practices

  • Enforce strong password policies
  • Use MFA wherever possible
  • Limit token lifetime and scope
  • Log and monitor authentication events

8. Challenges

  • Credential theft and phishing attacks
  • Session hijacking and replay attacks
  • Managing access across distributed systems

9. Applications

  • Enterprise identity management
  • Cloud access control
  • API security and token-based access
  • Secure login for web and mobile apps
