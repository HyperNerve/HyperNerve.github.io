System Security

System Security focuses on protecting individual computing systems—such as desktops, servers, and mobile devices—from unauthorized access, malware, and exploitation. It includes securing operating systems, applications, and user environments.

1. Objectives

  • Prevent unauthorized access to system resources
  • Protect against malware and vulnerabilities
  • Ensure system integrity and uptime

2. Threats to System Security

  • Privilege escalation
  • Rootkits and keyloggers
  • Unpatched software vulnerabilities
  • Malicious scripts and executables
  • Unauthorized configuration changes

3. Security Measures

  • Authentication: Passwords, biometrics, multi-factor
  • Access Control: Role-based permissions, file system restrictions
  • Patch Management: Regular updates to OS and software
  • Antivirus & Anti-malware: Detect and remove threats
  • System Hardening: Disable unused services, enforce policies

4. Example: Securing a Linux Server

Steps:
  - Disable root login via SSH
  - Use key-based authentication
  - Install and configure firewall (e.g., iptables or ufw)
  - Apply regular security updates
  - Monitor system logs with auditd

5. Logging & Auditing

  • Track login attempts and system changes
  • Use centralized logging tools (e.g., syslog, journald)
  • Audit file access and permission changes

6. Backup & Recovery

  • Automated backups of critical data
  • Disaster recovery plans
  • Snapshot and rollback mechanisms

7. Best Practices

  • Use least privilege principle
  • Encrypt sensitive files and partitions
  • Disable unnecessary services and ports
  • Regularly scan for vulnerabilities

8. Challenges

  • Balancing usability with security controls
  • Securing legacy systems and software
  • Detecting stealthy or persistent threats

9. Applications

  • Enterprise workstation and server protection
  • Secure configuration of embedded systems
  • Mobile device management
  • Virtual machine and container security
