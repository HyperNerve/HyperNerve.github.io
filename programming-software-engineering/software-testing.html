Software Testing

Software testing is the process of evaluating a software system to detect differences between expected and actual behavior. It ensures reliability, correctness, and performance before deployment.

1. Goals of Software Testing

  • Verification: Ensure software meets specifications
  • Validation: Ensure software fulfills user needs
  • Defect Detection: Identify and fix bugs
  • Quality Assurance: Improve overall software quality

2. Types of Testing

  • Manual Testing: Human testers execute test cases
  • Automated Testing: Scripts and tools perform tests

3. Levels of Testing

  • Unit Testing: Test individual functions or modules
  • Integration Testing: Test interactions between modules
  • System Testing: Test complete system behavior
  • Acceptance Testing: Validate system against user requirements

4. Testing Techniques

  • Black Box Testing: Test without knowledge of internal code
  • White Box Testing: Test with full knowledge of internal logic
  • Grey Box Testing: Partial knowledge of internal structure

5. Test Case Example

Function: add(a, b)
Test Case:
  Input: a = 2, b = 3
  Expected Output: 5

6. Unit Testing Example (Python)

def add(a, b):
    return a + b

import unittest

class TestMath(unittest.TestCase):
    def test_add(self):
        self.assertEqual(add(2, 3), 5)

unittest.main()

7. Test Coverage

  • Statement Coverage: Every line of code is executed
  • Branch Coverage: Every decision path is tested
  • Path Coverage: All possible execution paths are tested

9. Testing Tools

  • JUnit: Java unit testing framework
  • PyTest: Python testing framework
  • Selenium: Web application testing automation

10. Best Practices

  • Write Clear Test Cases: Define inputs, outputs, and conditions
  • Automate Repetitive Tests: Use scripts for efficiency
  • Test Early and Often: Catch bugs early in development
  • Use Continuous Integration: Run tests automatically on code changes
