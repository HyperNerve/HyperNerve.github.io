Discrete Mathematics

Discrete Mathematics is the study of mathematical structures that are fundamentally discrete rather than continuous. It provides the theoretical foundation for many areas of computer science, including algorithms, data structures, logic, and automata theory.

1. Logic

Propositional Logic

Deals with statements that are either true or false. Uses logical connectives like AND (∧), OR (∨), NOT (¬), IMPLIES (→).

Example: "It is raining AND it is cold" is true only if both conditions are true.

Predicate Logic

Extends propositional logic by including quantifiers and variables. Useful for expressing statements about objects.

Quantifiers: ∀ (for all), ∃ (there exists)

Example: ∀x (x > 0 → x² > 0)

2. Set Theory

Set

A collection of distinct elements. Can be finite or infinite.

Example: A = {1, 2, 3}

Set Operations

  • Union (A ∪ B): All elements in A or B
  • Intersection (A ∩ B): Elements common to both
  • Difference (A − B): Elements in A but not in B
  • Complement: Elements not in the set

Power Set

The set of all subsets of a set.

Example: Power set of {a, b} is { {}, {a}, {b}, {a, b} }

3. Relations and Functions

Relation

A relation is a set of ordered pairs. It defines a relationship between elements of two sets.

Example: R = {(1,2), (2,3)} is a relation from set A to B.

Properties of Relations

  • Reflexive: (a, a) ∈ R for all a
  • Symmetric: If (a, b) ∈ R then (b, a) ∈ R
  • Transitive: If (a, b) ∈ R and (b, c) ∈ R then (a, c) ∈ R
  • Equivalence Relation: Reflexive, symmetric, and transitive

Function

A relation where each input has exactly one output.

Types: Injective (one-to-one), Surjective (onto), Bijective (both)

Example: f(x) = x² is a function from ℝ to ℝ⁺

4. Combinatorics

Permutation

An arrangement of elements in a specific order.

Formula: n! (n factorial)

Example: 3 elements → 3! = 6 permutations

Combination

A selection of elements without regard to order.

Formula: nCr = n! / (r!(n−r)!)

Example: Choosing 2 out of 4 items → 4C2 = 6

Pigeonhole Principle

If n items are put into m containers and n > m, then at least one container has more than one item.

Example: 13 socks in 12 drawers → at least one drawer has 2 socks.

Inclusion-Exclusion Principle

Used to count the number of elements in the union of overlapping sets.

Formula: |A ∪ B| = |A| + |B| − |A ∩ B|

5. Graph Theory

Graph

A graph is a set of vertices (nodes) and edges (connections).

Types: Directed, Undirected, Weighted, Unweighted

Path

A sequence of edges connecting vertices.

Cycle

A path that starts and ends at the same vertex.

Connected Graph

There is a path between every pair of vertices.

Tree

A connected graph with no cycles. A special kind of graph.

Degree

Number of edges incident to a vertex.

Eulerian Path

A path that visits every edge exactly once.

Hamiltonian Path

A path that visits every vertex exactly once.

6. Number Theory (in CS)

Prime Numbers

Numbers greater than 1 that have no divisors other than 1 and itself.

Modular Arithmetic

Arithmetic system for integers where numbers wrap around after reaching a certain value (modulus).

Example: 17 mod 5 = 2

Greatest Common Divisor (GCD)

The largest number that divides two integers without a remainder.

Least Common Multiple (LCM)

The smallest number that is a multiple of two integers.

7. Applications in Computer Science

  • Logic: Used in circuit design, programming languages, and AI.
  • Set Theory: Used in database queries and data modeling.
  • Relations: Used in relational databases and graph modeling.
  • Combinatorics: Used in algorithm analysis and probability.
  • Graph Theory: Used in networking, social media, and pathfinding.
  • Number Theory: Used in cryptography and hashing.
