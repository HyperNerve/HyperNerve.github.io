Formal Language Theory

Formal Language Theory studies the structure, classification, and properties of languages defined over alphabets. It provides the basis for automata theory, compiler design, and syntactic analysis.

1. Basic Definitions

  • Alphabet (Σ): A finite set of symbols. Example: Σ = {a, b}
  • String: A finite sequence of symbols from Σ. Example: "abba"
  • Empty String (ε): A string with zero symbols.
  • Language: A set of strings over an alphabet. Example: L = {aⁿbⁿ | n ≥ 0}

2. Operations on Languages

  • Union (L₁ ∪ L₂): All strings in either L₁ or L₂
  • Concatenation (L₁L₂): All strings formed by concatenating a string from L₁ with one from L₂
  • Kleene Star (L*): All strings formed by concatenating zero or more strings from L
  • Intersection (L₁ ∩ L₂): Strings common to both languages
  • Complement (Σ* − L): All strings over Σ not in L

3. Grammar Types

Formal Grammar

A formal grammar G = (V, Σ, R, S) consists of:

  • V: Set of variables (non-terminals)
  • Σ: Set of terminals
  • R: Set of production rules
  • S: Start symbol

Example Grammar

S → aSb | ε

Generates strings like "ab", "aabb", "aaabbb", etc.

4. Chomsky Hierarchy

Type Language Class Grammar Type Automaton
Type 0 Recursively Enumerable Unrestricted Grammar Turing Machine
Type 1 Context-Sensitive Context-Sensitive Grammar Linear Bounded Automaton
Type 2 Context-Free Context-Free Grammar Pushdown Automaton
Type 3 Regular Regular Grammar Finite Automaton

5. Regular Languages

  • Defined by regular expressions or regular grammars
  • Recognized by DFA/NFA
  • Closed under union, intersection, complement, concatenation, and Kleene star

6. Context-Free Languages

  • Generated by context-free grammars
  • Recognized by pushdown automata
  • Can describe nested structures (e.g., parentheses)
  • Not closed under intersection or complement

7. Context-Sensitive Languages

  • Generated by context-sensitive grammars
  • Recognized by linear bounded automata
  • Can describe more complex patterns like aⁿbⁿcⁿ

8. Recursively Enumerable Languages

  • Generated by unrestricted grammars
  • Recognized by Turing machines
  • Includes all computable languages

9. Ambiguity in Grammars

Definition

A grammar is ambiguous if there exists a string with more than one leftmost derivation or parse tree.

Example:

E → E + E | E * E | id

The string "id + id * id" has multiple parse trees.

Implications

Ambiguity complicates parsing and compiler design. Some languages are inherently ambiguous.

10. Applications

  • Compiler Design: Lexical and syntax analysis
  • Natural Language Processing: Parsing sentences using grammars
  • Formal Verification: Modeling system behavior with formal languages
  • Programming Languages: Defining syntax using context-free grammars
